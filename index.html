UI Components

Performance

React Native

  • A compelling reason for using React Native instead of WebView-based tools is to achieve 60 frames per second and a native look and feel to your apps
  • React Native will render using native components
  • These design patterns are imposed by the operating system (components like Listview, Picker, Toolbar, etc..). It will give the better performance and smoother animations
  • Component explorer is available in https://play.google.com/store/apps/details?id=org.saravanakumar.reactnativecomponentsexplorer

Ionic

  • Components will render using HTML and CSS in WebView
  • Hence, The User Experience and Performance is less than Native components (React Native)
  • Component explorer is available in https://play.google.com/store/apps/details?id=com.ionichelper.ionic2components
Availability & Usage

React Native

  • Mostly all required components are available
  • Only a few Components (like NavigatorIOS, DatePickerIOS, DrawerLayoutAndroid, etc..) not available for all Platforms. As the name implies, components are only available on Android/iOS. Need to use third-party cross-platform solution in JavaScript. Read more...
  • By default Platform specific themes are not available (like Material style for android and iOS. Need to use the third-party Cross-Platform solution like as below:
    • NativeBase
    • React Native Elements
  • Platform-specific components need to implement like as below. And needs to test it properly 
        if (Platform.OS === 'ios') {
      AlertIOS.alert(title, message, buttons);
    } else if (Platform.OS === 'android') {
      AlertAndroid.alert(title, message, buttons, options);
    }

Ionic

  • Mostly all required components are available
  • Component themes are loading by HTML and CSS. Themes like Material for Android, iOS, and Windows supported from Ionic by default
  • CSS is managed by SASS and very easy to customize

Native Features

API Access

React Native

  • Native features such as Camera, Share, online/offline status, etc supported by default
  • Native features are easily testable by Emulators

Ionic

  • Ionic Native is a TypeScript wrapper for Cordova/PhoneGap plugins that make adding any native functionality
  • Needs some additional configuration or Native features not possible to test in browsers. Need to install an app in Emulator/Device

Offline

React Native

  • On iOS, AsyncStorage is backed by native code that stores small values in a serialized dictionary and larger values in separate files
  • On Android, will use either RocksDB or SQLite based on what is available
  • Redux Persist to store fetched data in AsyncStorage

Ionic

  • Cordova has cordova-sqlite-storage plugin
  • When running on the web or as a Progressive Web App, Storage will attempt to use IndexedDB, WebSQL, and localstorage, in that order

Running, Testing & Debug

Running

React Native

  • As this is the real native application, It can be run only in mobile simulators like iOS and Android. We cannot test in any browsers
  • Expo helps developers to build and share truly native apps. It syncs using QR code, your mobile and computer should be in the same wireless network

Ionic

  • As this is WebView we can run in browsers (like chrome, safari)
  • For Testing, deployment, Package and Error monitor Various products are available from Ionic Pro (Drifty co)
Debug

React Native

  • We can use the standalone version of React Developer Tools to debug the React component hierarchy.
  • Fast Hot reload after every changes (under 1 second) in emulator/device

Ionic

  • Easily can test in Browser
  • Fast Hot reload in browser
  • Debug in browser Developer tools
Unit & E2E Testing

React Native

  • Jest from Facebook
  • E2E detox from Wix

Ionic

  • As It's just an Angular app, This is Angular Testing, not Ionic
  • The Jasmine test framework provides everything needed to write basic tests. It ships with an HTML test runner that executes tests in the browser
  • Angular testing utilities create a test environment for the Angular application code under test. Use them to condition and control parts of the application as they interact within the Angular environment
  • The karma test runner is ideal for writing and running unit tests while developing the application. It can be an integral part of the project's development and continuous integration processes. This guide describes how to set up and run tests with karma
  • Use protractor (from angular) to write and run end-to-end (e2e) tests. End-to-end tests explore the application as users experience it. In e2e testing, one process runs the real application and a second process runs protractor tests that simulate user behavior and assert that the application responds in the browser as expected
